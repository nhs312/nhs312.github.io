Help -> Install New Software 에 들어간 뒤 Add를 누르고 새로운 URL을 추가한다.
이름은 뭐 아무렇게나 지어도 상관없는데 보기 쉽게 subclipse로 지어주고
http://subclipse.tigris.org/update_1.8.x/
이 주소를 추가 해주면 잠시 pending을 거친 뒤에 목록이 뜬다.
모두 체크해주고
  • Subclipse Ingetration for Mylyn 3.x (Optional)
  • Subversion Revision Graph
두 항목은 설치하지 않는다.
그리고 next - next next next.. 그러면 subclipse는 설치가 끝났다.

svn을 사용하기 위해서 JavaHL Library를 설치해야 한다.

다음 주소는 subclipse 공식 wiki이므로 참조한다.
http://subclipse.tigris.org/wiki/JavaHL
나는 이미 Homebrew를 설치한 상태이기 때문에 터미널을 열고 homebrew command를 입력한다.
(Macports를 설치한 사람은 Macports에 맞게 입력..)
brew install --universal --java subversion
이렇게 입력하고 나면 1분정도 시간이 걸리고 설치가 완료된다.
<터미널에서 homebrew를 통해 설치완료>
이런식으로 완료 메세지가 나오면 성공
그리고서 aptana에 들어가서 new project -> svn -> svn address 로 진행하다 보면 막힌다!?
These are the errors that were encountered:
no libsvnjavahl-1 in java.library.path
no svnjavahl-1 in java.library.path
no svnjavahl in java.library.path
java.library.path = .:/Library/Java/Extensions:/System/Library/Java/Extensions:/usr/lib/java
이런 에러 메세지가 뜨면서 제대로 불러오지 못하는 상황이 발생할 수 있다.
나는 이걸 해결하느라 하루가 넘게 걸렸는데 누군가 이 글을 본다면.. 그러지 않기를..ㅜㅜ

Preferences -> Team -> SVN 에 들어가서 SVN Interface를 SVN kit으로 바꿔준다.
<하단의 SVN interface를 바꾸자>
그러면 svn을 제대로 이용할 수 있다.

아 검색하다보니까 저 에러메세지에 대한 이상한 자료에 낚여서 시간을 많이 날렸다 ㅜㅜ
